# Requirements — 타미야 대회 일정 뷰어

## 서비스 개요
- 핵심 가치 제안: 타미야 코리아 매장·공인경기장 대회 일정을 크롤링해 종목별로 분리·정리한 리스트로 보여주는 웹앱
- 주요 사용자: 타미야 미니카 레이싱 참가자
- 핵심 시나리오
  1. 앱 접속 → 최신 대회 일정 자동 로드 → 날짜·종목·장소별 리스트 확인
  2. 슬래시(/) 구분 종목을 개별 행으로 분리해 확인
  3. 종목 필터로 원하는 클래스만 보기

## 크롤링 대상
- URL: `https://tamiya.co.kr/bbs/board.php?bo_table=club_race&ser=0`
- 필드 매핑:
  - `td.category` [0번째] → 대회명
  - `td.category` [1번째] → 장소
  - `td.date` → 날짜 (YYYY.MM.DD 형식)
  - `td.event` → 시간+종목 문자열 (` / ` 구분자로 여러 종목 포함 가능)

## 종목 분리 규칙
- `td.event` 값: `"13:00 M2 클래스 / 15:00 M3 클래스"`
- ` / ` 로 split → 각 항목을 독립 행으로 표시
- 각 항목 형식: `HH:MM 종목명`

## 기능 요구사항

### Must Have
- [ ] 서버사이드 크롤링 API (`/api/races`) — CORS 우회 목적
- [ ] 대회 목록 리스트 뷰 (날짜순 정렬)
- [ ] 종목 `/` 분리 후 개별 행 표시
- [ ] 날짜 표시 (YYYY.MM.DD → M월 D일 한국어)
- [ ] 대회명, 장소, 시간, 종목 4개 컬럼
- [ ] 새로고침(재크롤링) 버튼

### Should Have
- [ ] 종목 필터 (M1/M2/M3/M2B/OPEN 등)
- [ ] 날짜별 그룹핑
- [ ] 과거 대회 구분 (날짜 기준 dim 처리)
- [ ] 반응형 (모바일 테이블 → 카드)

### Could Have
- [ ] 특정 종목 즐겨찾기
- [ ] 캘린더 뷰

## 비기능 요구사항
- 크롤링: 서버사이드 Node.js proxy (CORS 우회)
- 캐싱: 10분 TTL (타미야 사이트 부하 방지)
- 성능: 첫 로드 2초 이내
- 반응형: 모바일(360px)~데스크탑

## 화면 목록

## 기술 구조
- **Frontend**: React SPA (Vite)
- **Backend**: Node.js Express 프록시 서버 (크롤링 담당)
  - `GET /api/races` → HTML 파싱 → JSON 반환
- **크롤링 라이브러리**: cheerio (서버사이드 HTML 파싱)
